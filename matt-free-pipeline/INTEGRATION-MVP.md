# Matt-free backbone → apworld MVP integration

Branch: `feat/matt-free-backbone-mvp`. Goal: generate + boot a **Shattering** seed whose location
set comes from the data-derived, matt-free backbone (not matt's location list).

## What this wires
- **Location source**: `Archipelago/worlds/eldenring/matt_free_locations.py` — 3,944 checks across
  161 region keys (34 populated), keyed to the apworld's own `region_order` names. Generated by
  `matt-free-pipeline/gen_mattfree_locations.py` from `region_map.csv`.
- **Client detection**: `Archipelago/worlds/eldenring/er_static_detection_table_mattfree.json`
  = `{ap_code: [flag]}` built from the backbone flags. Each location carries a synthetic
  `key="mf:<flag>"` (matt-free) so the apworld's `locationFlags` slot_data path emits it.
- **Access model**: none needed — the apworld already implements the Shattering as
  `world_logic = region_lock` (region reachable iff you hold its `REGION_LOCK_ITEM`; Roundtable Hold
  is the free hub). Just set it in the yaml.

## Scoped OUT of the MVP (per Alaric)
- Quest/NPC `global` items (621), scattered-filler `global_filler` (spam), phantom `shop_reference`
  (20 — items placed elsewhere). Questlines are derandomized, so those slots aren't checks.

## Gen steps (Windows — the sandbox can't run AP gen)
The patch is BYTE-SAFE + self-verifying (aborts + restores on a bad write). Toggle model:
`--apply` = matt-free active, `--revert` = normal. No env var. It edits ONE line of `__init__.py`
(location-source override) and SWAPS the detection JSON file; `slot_data.py` is NOT touched.
1. `python patch_use_mattfree_backbone.py --check`   (anchors + files intact; run from repo root)
2. `python patch_use_mattfree_backbone.py --apply`
3. Generate with a `world_logic: region_lock` yaml (see `EldenRing-mattfree-mvp.yaml`, or set that key
   on your known-good Alaric.yaml; `enable_dlc: true` to populate DLC regions):
   ```
   .\build.ps1 -Generate
   ```
4. Back to normal apworld: `python patch_use_mattfree_backbone.py --revert`

## If a previous (buggy) apply corrupted __init__.py / slot_data.py
The old text-mode patch collapsed those files to one line. Restore then re-apply with the byte-safe patch:
```
git checkout -- Archipelago/worlds/eldenring/__init__.py Archipelago/worlds/eldenring/slot_data.py
del Archipelago\worlds\eldenring\__init__.py.bak_mattfree Archipelago\worlds\eldenring\slot_data.py.bak_mattfree
python patch_use_mattfree_backbone.py --check   # should say OK
python patch_use_mattfree_backbone.py --apply
```

## Known limitations (this is a FIRST CUT — expect iteration on gen)
- **Untested gen.** Built + structurally verified in the sandbox (module imports, 161 region keys so
  `create_regions[region]` can't KeyError, ap_codes/flags unique, detection table consistent, patch
  py_compiles + anchors present) but NOT gen-tested — AP needs Python 3.11+ on your box.
- **Coarse region lumping.** DLC largely collapses into `Gravesite Plain` (594) and undergrounds into
  `Nokron, Eternal City Start` (398); `shop_multi` (multi-merchant) + Roundtable stock sit in
  `Roundtable Hold` (always-open). Fine for a first seed; refine `REGION_MAP`/`PLAY2AP` in the
  generator to split them.
- **Overworld** tiles placed via grace/BonfireWarp anchors + nearest-neighbor (627 direct, 308 NN @
  median 1 tile) — good, but a handful of far-NN tiles may be off.
- **Item pool**: 342 locations whose backbone default item isn't a registered `items.py` name fall
  back to `Golden Rune [1]`. Only affects the vanilla-default pool contribution, not reachability.
- If gen FillErrors, likely causes: a region with progression-only demand but too few checks, or the
  item pool count vs location count. First lever: widen accessibility / check the goal (`ending_condition`).

## Reproduce / tweak
`python matt-free-pipeline/gen_mattfree_locations.py` (edit `REGION_MAP` / `PLAY2AP` at the top to
refine region splits), then re-copy `matt_free_locations.py` + `er_static_detection_table_mattfree.json`
into `Archipelago/worlds/eldenring/`.
